Clock

Hardware clock module detected

A hardware clock module is providing the time to the system instead of an internet time server. If you unplug the module, or if you disable it in the settings of the Power Settings addon, then the system will fall back to your internet time preference instead.

The hardware clock's time was initially set with data from an internet time server. If the clock is out of sync you can re-sync it from the internet. Or you could provide the time manually.

Internet time

If your system does not have a hardware clock module installed, every time the system starts it will try to get the current date and time from an internet server.

If enabled, as soon as there is an internet connection, any manually provided time will be overridden by an internet NTP server.



Manually set the time

Most smart home devices connect to the internet get the current time. This can share your IP address with third parties who provide that service. Since nothing is free, you might want to set the time manually instead. However, you will have to do this every time you reboot.

You can manually set the time of the hardware clock.

If you're trying to change the time to match your timezone, then please first make sure your timezone is set correctly under localisation.

Security

Secure access

You are currently accessing your Controller without using an encrypted connection.

While there shouldn't be any devices maliciously listening in on your home network, it's best to play it safe, and connect more securely.

All this requires is replacing http with https in the current URL. The extra S stands for 'secure'.

http://candle.local → https://candle.local

This will unfortunately result in some warnings that the security certificate is 'self-signed', which you will have to choose to ignore, and 'connect anyway'.

An example of allowing an encrypted connection

Why do I get those warnings?

The short explanation for this is that browsers rightfully warn you about 'self-signed' security certificates. On the internet a certificate guarantees that the URL you are visiting really is who they say they are, and that there's not some other server "in the middle" pretending to be the destination.

But there is, unfortunately and shamefully, no standardized method for creating such secure connections inside local networks. Encrypted connections on local networks are given the same scary warning as a server on the internet would get, even though it's much less likely that there's a malicious middleman on your local network.

This situation is part of why it has become common practise to connect smart home devices to 'the cloud', even though the device you're trying to connect to is on your local network. It's also disincentivising creating cloudless solutions, such as Candle.

A round-trip via the cloud is considered normal...

...while connecting directly results in scary warnings.

If for some reason you wish to go to the non-secure version, you can do so by by opening the URL with /settings appended to it. This will skip the "upgrade to https security" process.

Install a security certificate (optional)

If you install a 'security certificate' from this controller, then your browser will trust it completely from now on.

It's a more permanent solution, but the warnings you'll get are even scarier.

This will replace your existing:

  • Settings for Candle addons
  • General settings for controller and your addons
  • Data logs (if present in the backup)

Troubleshooting

Disable addons

If you suspect that a specific addon is causing trouble, you can disable it here.

Restore original addons

If for some reason one of the original addons isn't loading or working properly, then you can choose to revert to a backup of the originally installed version.

You also have the option to delete an addon's stored settings data. This will cause the addon to regenerate it's default settings the next time it starts.

When you revert, you should probably do both, so that the settings data won't be for a later version of the addon. Such a version mismatch could cause issues.

Either option will first (attempt to) disable the current version of the addon.

    It's wise to reboot the controller after reverting an addon to complete the process.

    After the reboot you'll have to go to the Candle store to manually enable the addon again. Only the Candle Store itself is automatically re-enabled at boot, since it's vital to managing all other addons.

    You should also refresh the page once the old version is restored.

    Safe mode

    When your candle Controller boots into 'safe mode', only the original versions of three addons will seem to be installed:

    • Candle store
    • Candle theme
    • Power settings

    While in safe mode any misbehaving addons will no longer be loaded, and thus won't affect your controller. You can then try to solve any issues with them. For example, you can then manually disable addons, so they won't be a problem anymore when you boot back into normal mode.

    Important tip: you can always enable safe mode manually by placing a text file called 'candle_safe_mode.txt' on the controller's SD card.

    Re-install Candle App Store

    If for some reason your Candle Store still isn't working properly you can try switching to a different version.

    A safe option is to go back to the original version that came with your current version of Candle.

    Another option is to try and install the very latest "cutting edge" version that's available online. This version can take a little longer to install, since it's "built from scratch" on your device from the Candle Store's source code. This requires patience and an internet connection.

    This will automatically restart your controller and reload this page once complete.

    How a system update works

    Your Candle Controller has three main partitions: two 'system partitions', and one 'user partition'. Think of them as large chunks of storage space.

    Your installed addons and personal data is on the user partition, and is perfectly safe there. The update process doesn't touch it.

    The version of Candle you're using is running on the A system partition. The other system partition, B, is not being used.

    When a big Candle update is installed, the new version is downloaded to the inactive B partition. Once the download is complete you can try to switch over to the B system partiton. If that's succesful, then the B partition becomes the active one.

    And so on

    Then if there's another update available, then it will be downloaded to whichever partition is inactive. And so the system keeps switching between A and B.

    The nice thing about this "A-B method" is that it's very robust and safe. If switching to the new version fails for some reason, then the system automatically falls back to using the 'old' version.
